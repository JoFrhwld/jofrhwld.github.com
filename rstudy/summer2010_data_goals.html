What is a good solution?

When it comes to producing solutions for our problems in R, I think it would be good if we had a definition of what a good solution is. Here are some points that immediately came to mind.

  • It should be implementable in as few lines of code as possible.
  • Those lines of code should be as clear as possible.
  • It should produce as few new objects as possible.
  • It should be as general as possible.
